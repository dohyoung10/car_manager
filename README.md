# 🚗 내 차 정비 관리 시스템

## 1. 프로젝트 제목
내 차 정비 관리 시스템 (My Car Maintenance Manager)

## 2. 주제 설명
차량의 정비 기록을 일일이 기억하지 않아도 되도록, 부품별 교체 이력을 저장하고
**다음 교체 예정일과 남은 기간(D-day)을 자동으로 계산**해 알려주는 웹 서비스입니다.
회원가입/로그인으로 개인별 차량을 안전하게 관리하며, 관리자 페이지에서 전체
회원과 차량을 관리할 수 있습니다.

## 3. 주요 기능
- **회원 기능**: 회원가입 / 로그인 / 로그아웃 (비밀번호 암호화 저장)
- **차량 관리(CRUD)**: 차량 등록 / 목록 조회 / 삭제
- **정비기록 관리(CRUD)**: 부품별 기록 등록 / 조회 / 수정 / 삭제
- **자동 알림 기능(핵심)**: 마지막 교체일 + 주기로 다음 교체일·D-day 자동 계산,
  상태를 색상(빨강=지남 / 주황=곧 / 초록=여유)으로 표시
- **관리자 페이지**: 전체 회원·차량 조회, 회원 삭제 (admin 권한만 접근)

## 4. DB 구조 (SQLite, car.db)
- **users** (id, username, password, role) — 회원 정보, role로 일반/관리자 구분
- **cars** (id, user_id, car_name, mileage) — 차량 정보 (users와 1:N 관계)
- **records** (id, car_id, part_name, last_date, cycle_months, memo) — 정비기록 (cars와 1:N 관계)

## 5. 느낀 점
처음에는 라우트와 DB 연결이 어떻게 이어지는지 막막했지만, 게시판 예제를 분석하면서
"요청 → 라우트 함수 → SQL 실행 → 템플릿 렌더링" 흐름을 이해하게 되었다.
특히 날짜 계산으로 D-day를 만드는 부분을 직접 구현하면서, 단순 저장을 넘어
데이터를 '가공해서 보여주는' 것이 서비스의 핵심이라는 걸 배웠다. 비밀번호를
평문이 아니라 해시로 저장하는 보안 처리도 새로 알게 된 부분이다.
